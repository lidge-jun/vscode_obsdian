## Part 1 - Easy Explanation

The current HWP screen can show the document, but it trusts an internet iframe
too much and shows editing/saving before VS Code can safely protect the file.

Phase 8.2d makes it safer in one implementation pass:

1. Put the HWP engine inside the extension package instead of loading it live
   from GitHub Pages.
2. Lock the webview with CSP so only extension-owned resources run.
3. Replace loose messages with named, checked HWP messages.
4. Make HWP/HWPX viewer-only by default.
5. Keep save/export behind an explicit experimental setting, with collision
   checks and atomic writes.

Simple version: first we remove the unsafe door, then we add a lock, then we
hide the save button until the full editor lifecycle is ready.

## Implementation Policy

### Default feature level

```text
Default: HWP/HWPX viewer-only.
Experimental: save/export hidden unless vscode-obsdian.hwp.experimentalSave is true.
Not implemented in this phase: full editable CustomEditorProvider lifecycle.
```

Reason:

- The current provider is `CustomReadonlyEditorProvider`.
- VS Code dirty state, Cmd+S, Save All, backup, revert, and hot-exit are not
  wired.
- Shipping visible editing as default would still be dishonest even after bridge
  hardening.

### Runtime source

```text
Default runtime: extension-local rhwp-studio bundle.
Blocked default runtime: https://edwardkim.github.io/rhwp/
```

The B phase must create `vendor/rhwp-studio-dist/VERSION.md` with:

```text
upstream: https://github.com/edwardkim/rhwp
npm wrapper: @rhwp/editor@0.7.13
source commit/tag: <exact commit or tag used>
bundle command/source: <exact command or immutable artifact URL>
notes: external live GitHub Pages is not used at runtime
```

If upstream does not expose a reproducible local studio dist, B must stop and
report this as a blocker instead of silently falling back to the live remote URL.

## File Plan

### NEW `src/common/hwpMessageSchema.ts`

Purpose:

- Namespace HWP events.
- Runtime-validate only HWP payloads.
- Share payload types between extension host and React.

Expected complete content shape:

```typescript
export const HWP_EVENTS = {
    init: 'hwp:init',
    fileData: 'hwp:fileData',
    requestSave: 'hwp:requestSave',
    saveResult: 'hwp:saveResult',
    error: 'hwp:error',
} as const;

export type HwpEventName = typeof HWP_EVENTS[keyof typeof HWP_EVENTS];

export interface HwpFileDataPayload {
    fileName: string;
    buffer: number[];
    fileSize: number;
    isHwpx: boolean;
    error?: string;
}

export interface HwpSavePayload {
    bytes: number[];
    sourceFileName: string;
    isHwpx: boolean;
}

export interface HwpSaveResultPayload {
    success: boolean;
    savedPath?: string;
    convertedFromHwpx?: boolean;
    error?: string;
}

export interface HwpErrorPayload {
    error: string;
}

export function isHwpEvent(value: unknown): value is HwpEventName;
export function validateHwpPayload(type: HwpEventName, payload: unknown): boolean;
```

Validation rules:

- `buffer` and `bytes` must be arrays of integers from 0 to 255.
- `fileSize` must be a non-negative number.
- save payload must not be accepted if bytes are empty.
- unknown event names are not HWP events and are ignored by this schema.

### NEW `src/react/view/hwp/rhwpBridge/validateRhwpMessage.ts`

Purpose: validate iframe response envelopes before resolving requests.

Expected behavior:

```typescript
export function validateRhwpResponse(value: unknown): RhwpResponse | undefined {
    if (!value || typeof value !== 'object') return undefined;
    const record = value as Record<string, unknown>;
    if (record.type !== 'rhwp-response') return undefined;
    if (typeof record.id !== 'number') return undefined;
    if ('error' in record && typeof record.error !== 'string') return undefined;
    return record as RhwpResponse;
}
```

The validator must reject unrelated `window.message` traffic.

### NEW `src/react/view/hwp/rhwpBridge/createSecureRhwpEditor.ts`

Purpose:

- Create the local rhwp iframe.
- Verify `event.source === iframe.contentWindow`.
- Verify `event.origin === new URL(studioUrl).origin`.
- Use `postMessage(..., expectedOrigin)`, not `'*'`.
- Use one removable `message` listener.
- Use 30s default timeout.
- Keep `loadFile` protocol compatible with `@rhwp/editor@0.7.13` by sending
  `Array.from(data)`.

## Business Logic Decisions

Default decisions for this phase:

1. HWP/HWPX is viewer-only by default.
2. Experimental save is opt-in and is not advertised as normal editing.
3. `.hwpx` export target defaults to `basename.converted.hwp`.
4. Existing target files are never overwritten without explicit confirmation.
5. If local rhwp-studio cannot be reproduced or bundled, B stops instead of
   falling back to a remote iframe.
