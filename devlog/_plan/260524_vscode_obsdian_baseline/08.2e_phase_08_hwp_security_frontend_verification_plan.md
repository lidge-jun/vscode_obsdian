## Verification Plan

Run in B/C:

```text
git diff --check
./node_modules/.bin/tsc --noEmit
./node_modules/.bin/tsc --noEmit -p src/react/tsconfig.json
npm run build
npm run package
unzip -l vscode-obsdian-*.vsix | rg 'extension/resource/rhwp-studio'
npm audit --omit=dev
```

CSP/WASM spike gate before claiming HWP runtime complete:

```text
1. Open the local rhwp-studio iframe in Extension Development Host.
2. Capture DevTools CSP errors while loading the known .hwpx fixture.
3. If WASM needs a relaxation, record the exact directive and reason.
4. Production CSP may add only evidenced directives such as wasm-unsafe-eval,
   worker-src blob:, or blob: for worker/runtime assets.
5. Production CSP must not add broad https:, *, or remote frame/connect origins.
6. Dev CSP must allow 127.0.0.1:5739 only for the Vite server.
```

Known audit policy:

- `npm audit --omit=dev` currently reports non-HWP dependency findings.
- If still failing, do not claim dependency audit clean.
- Update `98_dependency_audit_snapshot.md` with the new output and keep
  marketplace release blocked unless the user explicitly accepts that risk.

Manual VS Code Insiders QA with Computer Use:

```text
1. Install the newly packaged VSIX in VS Code Insiders.
2. Open the known failing .hwpx fixture.
3. Confirm no live https://edwardkim.github.io/rhwp/ network dependency is used.
4. Confirm the document viewport appears.
5. Confirm full-screen spinner stops after loaded or warning.
6. Confirm stale "Failed to load: Request timeout: loadFile" is not fatal red state.
7. Confirm Save HWP button is hidden by default.
8. Enable vscode-obsdian.hwp.experimentalSave only for save-path QA.
9. Confirm .hwpx save target cannot silently overwrite an existing .hwp.
10. Reopen PPTX and XLSX smoke files to catch webview CSP regressions.
```

## Employee Audit Targets For A Phase

- Verify local bundle path and CSP shape.
- Verify no default live remote iframe remains.
- Verify HWP message schema covers host receive and React receive.
- Verify save path cannot overwrite silently.

  missing `rhwpStudioUrl`, experimental save.
- Verify iframe bridge listener cleanup and timeout warning behavior.
- Verify toolbar/badge layout does not regress.

## Business Logic Decisions

Default decisions for this phase:

1. HWP/HWPX is viewer-only by default.
2. Experimental save is opt-in and is not advertised as normal editing.
3. `.hwpx` export target defaults to `basename.converted.hwp`.
4. Existing target files are never overwritten without explicit confirmation.
5. If local rhwp-studio cannot be reproduced or bundled, B stops instead of
   falling back to a remote iframe.
