# Phase 8.2f — HWP/HWPX Full Editing Recovery Audit

Date: 2026-05-29

Commit under audit:

```text

```

## Part 1 — Easy Explanation

The user rejected the safe-but-viewer-only HWP path. That correction is valid:
an HWP editor that shows the rhwp toolbar but disables save/editing is not useful
for the intended workflow.

The final direction is therefore:

```text
1. Use the full upstream rhwp studio by default.
2. Keep Save enabled by default.
3. Preserve HWP as HWP and HWPX as HWPX.
4. Warn before overwriting files.
5. Keep the local/offline hardening path as a later fallback, not the default.
```

This knowingly reopens part of the remote iframe trust tradeoff. The product
requirement is full rhwp functionality first; the local/offline hardening work
must not disable the editor by default again.

## What Went Wrong

### Attempt 1 — `@rhwp/editor` wrapper

The npm wrapper rendered documents through the upstream web app, but it used a
short request timeout. The VS Code webview could show the document while the
React wrapper still displayed a stale `loadFile` timeout. That made the UI look
broken even when rhwp had loaded.

Impact:

- false failure banner
- confusing spinner/error state
- no reliable save lifecycle

### Attempt 2 — local direct iframe

The extension tried loading the local `rhwp-studio` bundle through a VS Code
webview resource URI. In Insiders, that route produced a `chrome-error` iframe.

Impact:

- local/offline path did not render reliably
- full rhwp UI was not usable

### Attempt 3 — srcdoc local bundle bridge

The srcdoc bridge showed parts of the rhwp UI, but `loadFile` and export calls
timed out. This path also moved toward safe viewer-only behavior.

Impact:

- document load/export bridge was unreliable
- Save was disabled by default
- the UI showed `HWP/HWPX editing disabled`

### User Correction

The user clarified that viewer-only behavior defeats the purpose. The original
working behavior allowed broad rhwp editing despite spinner problems. The fix
therefore restores full rhwp behavior first and documents the remaining security
and lifecycle debt explicitly.

## Final Architecture

```text
.hwp/.hwpx file
  -> OfficeViewerProvider
  -> ReactApp route=hwp
  -> Hwp.tsx shell
  -> createSecureRhwpEditor()
  -> iframe src=https://edwardkim.github.io/rhwp/
  -> rhwp full editor UI
  -> exportHwp/exportHwpx
  -> hwp:requestSave
  -> hwpHandler modal overwrite guard
  -> atomic workspace.fs write
```

Key behavior:

- `vscode-obsdian.hwp.experimentalSave` defaults to `true`.
- `vscode-obsdian.hwp.studioUrl` defaults to
  `https://edwardkim.github.io/rhwp/`.
- HWP uses `exportHwp()`.
- HWPX uses `exportHwpx()`.
- HWP export must match OLE magic.
- HWPX export must match ZIP magic.
- Existing files are not silently overwritten.
- Disposed webview `postMessage` errors are logged instead of shown to users.

## File-Level Summary

### `package.json`

- Changed `vscode-obsdian.hwp.experimentalSave` default from `false` to `true`.
- Added `vscode-obsdian.hwp.studioUrl`.
- The default runtime is now the full upstream rhwp studio.

### `src/provider/officeViewerProvider.ts`

- Reads `hwp.experimentalSave` with default `true`.
- Reads `hwp.studioUrl` with default upstream URL.
- Passes `rhwpStudioUrl` to React.
- Stops injecting local `studioHtml` on the default path.

### `src/react/view/hwp/Hwp.tsx`

- Removes the default editing-disabled badge.
- Shows Save by default unless the setting is explicitly `false`.
- Uses `Save HWP` for `.hwp` and `Save HWPX` for `.hwpx`.
- Uses format-aware export.
- Treats load timeout as a warning path rather than a permanent editor block.

### `src/react/view/hwp/rhwpBridge/createSecureRhwpEditor.ts`

- Keeps the full remote iframe path available by default.
- Adds `exportHwpx()`.
- Extends iframe sandbox capabilities required by the full editor.
- Uses origin/source checks for remote iframe messages.
- Removes the fire-and-forget `loadFile` path that caused false success/failure.

### `src/provider/handlers/hwpHandler.ts`

- Enables save handler by default.
- Validates exported bytes by requested format.
- Saves `.hwpx` back as `.hwpx` by default.
- Keeps `.converted.hwp` only for explicit HWP conversion.
- Uses overwrite/choose/cancel prompts before writing.
- Writes through a temp file and rename.

### `src/common/handler.ts`

- Wraps outbound webview `postMessage`.
- Prevents disposed webview races from surfacing as user-facing errors.

### `src/common/reactApp.ts`

- Allows the upstream rhwp host in CSP `frame-src` and `connect-src`.
- Allows CDN assets needed by the upstream runtime.
