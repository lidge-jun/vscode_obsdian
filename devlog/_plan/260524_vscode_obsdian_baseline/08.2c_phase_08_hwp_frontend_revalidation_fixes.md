## Purpose

Frontend validation confirmed the backend/security plan is implementation-ready,
but found remaining WebView/Hwp.tsx ambiguity. This addendum closes those gaps.

## 1. 8.2A Scope Lock

8.2A is viewer safety only.

```text
8.2A does:
  - hide/disable save by default
  - replace HWPX save hint with viewer-only copy
  - show non-fatal warning when `loadFile` times out
  - keep spinner from staying forever after a warning path

8.2A does not:
  - detect late document success
  - poll page count
  - solve listener cleanup
  - solve origin/source validation
```

Late success recovery belongs to 8.2C through a React-local `onLoadStatus`
callback passed into `createSecureRhwpEditor`.

## 2. `onLoadStatus` Wiring

### Bridge emits load status

```typescript
type HwpLoadStatusPayload =
  | { status: 'loading' }
  | { status: 'loaded'; pageCount?: number }
  | { status: 'warning'; message: string }
  | { status: 'failed'; message: string };
```

### MODIFY `src/react/view/hwp/rhwpBridge/createSecureRhwpEditor.ts`

Add the callback to the bridge options:

```typescript
interface SecureRhwpEditorOptions {
  studioUrl: string;
  width: string;
  height: string;
  requestTimeoutMs?: number;
  onLoadStatus?: (status: HwpLoadStatusPayload) => void;
}
```

```diff
+  options.onLoadStatus?.({ status: 'loading' });
   try {
     const result = await request('loadFile', { data: Array.from(data), fileName });
+    options.onLoadStatus?.({ status: 'loaded', pageCount: result?.pageCount });
     return result;
   } catch (error) {
+    options.onLoadStatus?.({ status: 'warning', message: error instanceof Error ? error.message : String(error) });
     throw error;
   }
```

### MODIFY `src/react/view/hwp/Hwp.tsx`

```diff
+                    const editor = await createSecureRhwpEditor(containerRef.current, {
+                        studioUrl: rhwpStudioUrl,
+                        width: '100%',
+                        height: '100%',
+                        requestTimeoutMs: DEFAULT_RHWP_REQUEST_TIMEOUT_MS,
+                        onLoadStatus: (status) => {
+                            if (status.status === 'loaded') {
+                                setWarning(null);
+                                setLoading(false);
+                            } else if (status.status === 'warning') {
+                                setWarning(status.message);

+                            } else if (status.status === 'failed') {
+                                setError(status.message);
+                                setLoading(false);
+                            }
+                        },
+                    });
```

This callback is in-webview only. Do not send `loadStatus` through
`src/react/util/vscode.ts` or `src/common/handler.ts`; it does not cross the
extension-host trust boundary.

```diff
+request('loadFile', { data: Array.from(data), fileName })
```

This preserves the `@rhwp/editor@0.7.13` protocol exactly.

### MODIFY `validateRhwpMessage.ts`

```typescript
export function validateRhwpResponse(value: unknown): RhwpResponse | undefined {
  if (!value || typeof value !== 'object') return undefined;
  const record = value as Record<string, unknown>;
  if (record.type !== 'rhwp-response') return undefined;
  if (typeof record.id !== 'number') return undefined;
  if ('error' in record && typeof record.error !== 'string') return undefined;
  return record as RhwpResponse;
}
```

## 7. Frontend Acceptance Criteria

Frontend implementation is ready only when:

```text
1. Save is hidden unless hwpExperimentalSave is true.
2. HWPX save hint is hidden unless save is enabled.
3. Missing rhwpStudioUrl renders a fatal configuration Alert.
4. loadFile timeout warning does not keep fullscreen spinner active forever.
5. `onLoadStatus` has a direct bridge emit path and Hwp.tsx consume path.
6. createSecureRhwpEditor serializes loadFile bytes with Array.from(data).
7. requestTimeoutMs default is 30000.
8. React tsc can import the shared schema.
```
